Technical Program Manager · 9YR · Insurance/Manufacturing

일정만 관리하는 사람이 아니라
복잡한 운영을 끝까지 굴려내는 사람

보험·제조 도메인에서 ERP·CRM·FSM·정산 시스템 0→1을 끝까지 굴려낸 9년차 Technical Program Manager입니다. 여러 시스템과 외주·내부 조직이 얽힌 복잡한 환경에서 기술 의존성·리스크·운영 기준을 정렬하고, 제품 판단을 끝까지 굴러가는 시스템으로 남기는 방식으로 일합니다.

도메인은 바뀌어도 0→1을 끝까지 굴려내는 다섯 단계의 실행 패턴은 같았습니다. 좌측 lens에서 단계 칩을 누르면, 그 단계가 적용된 케이스와 패턴 카드만 강조됩니다.

01다섯 단계의 실행 패턴

0→1 프로젝트를 끝까지 굴려내기 위해 반복해서 적용한 다섯 단계입니다. 각 단계마다 실제 사용한 결정·발화·산출물을 같이 적었습니다.

P01

프로그램 구조화

복잡한 0→1 프로젝트를 입사 첫 주에 구조화합니다. 60명 TF를 직접 조직하고 6단계 로드맵·RFP·SRS·WBS를 첫 두 달 안에 일괄 세팅합니다.

"보안 이슈 해결(1순위) > 기존 운영 유지(1순위) > 사용성 개선(2순위)" — 입사 3일 차 명문화
Evidence
60명 TF 6단계 로드맵 RFP/SRS/WBS 일괄 세팅 ASP vs SI 시나리오 1차 범위 절단
P02

의존성 매핑

ERP·CMS·CRM·SAP가 동시에 움직이는 환경에서 시스템 간 인터페이스와 데이터 정합성을 설계합니다. 분리·수기로 운영되던 핵심 도메인을 하나의 정합성 안에 묶습니다.

항목 / 입력 / 기간 × 조직정보 · 인사정보 · 계약 · 사용인코드 · 상품목록 · 조정률 — 3개년 마이그레이션 매트릭스 표준화
Evidence
ERP-CMS-SAP 데이터 싱크 3개년 마이그레이션 매트릭스 수수료 검증 로직 3개 통합 ERP 5단계 영업 퍼널 정의
P03

외주·내부 R&R

외주 협업 프로토콜과 계약 구조를 운영 가능한 체계로 만듭니다. 그 패턴이 다음 외주에 그대로 재사용되도록 합니다.

유지보수 0.5M/월 풀 → 0.25M/월 + 별도 프로젝트 구조 재설계 · 노임단가 4단계 협상 (5,666,388원 → 8,000,000원)
Evidence
ASP 협업 프로토콜 5개 0.5M→0.25M 재설계 노임단가 4단계 협상 SAP·영선아이티 재사용 FSM 1년 유지보수 계약
P04

리스크·장애 통제

인프라·보안·장애 의사결정을 직접 합니다. 임시방편 차단 + 근본 원인 추적 + 재발 방지 3단계로 운영에 박습니다.

k8s + Redis + 로그 이중화 + AD/마스킹/VPN · GC Storm 재발 방지 3단계 (캐싱 / CDN / 부하 테스트) · 보안 코드리뷰 Path Traversal HIGH 우선순위
Evidence
k8s · Redis 인프라 AD · VPN · 마스킹 Path Traversal HIGH GC Storm 재발 방지 3단계 장애 재발률 0%
P05

운영 자산화

표준 프로토콜과 운영 기준이 다른 프로젝트에 재사용되어 조직 자산으로 남도록 합니다. 한 번의 프로젝트가 다음 프로젝트의 시작점이 됩니다.

ASP 협업 프로토콜 → SAP(BSG) · 영선아이티 후속 외주에 그대로 재사용 (framework 전이) · 표준 ERP 아키텍처가 후속 시스템 개발 기준으로 정착
Evidence
표준 ERP 아키텍처 ASP→SAP·영선 전이 4년+ 안정 운영 수기 업무 100% 제거 FSM 노하우 → SFA 이식

03대표 케이스

다섯 단계의 패턴이 실제로 어떻게 굴러갔는지 다섯 케이스로 보여드립니다.

Case 01 · 토스인슈어런스 · 2022.06 – 현재 · 4년+ 운영

보험 ERP 0→1 구축

보안 취약 시스템(MYGA)을 단순 교체한 게 아니라 60명 TF·3개년 마이그레이션·외주 프로토콜·인프라·보안까지 통합 ERP의 모든 레이어를 직접 설계했습니다.

PHP 5.2 + 그누보드 2016 기반의 MYGA가 보안팀 "수정·미티게이션 보완 불가" 판정을 받았고, 인사·조직·계약·상품·수수료·통계가 별도 시스템 + 엑셀로 분리·수기 관리되던 상태였습니다. 외주(지넥슨) 도입 결정만 있고 Why/Who/What/How는 정리되지 않았습니다. 1차 오픈 범위를 "MYGA 대체 + 기존 CRM 유지"로 잘라내고, 60명 TF 직접 조직 · 6단계 로드맵 · RFP/SRS/WBS 일괄 세팅 · ASP 채택 · 3개년 데이터 마이그레이션 매트릭스 표준화 · k8s/Redis 인프라 내재화 · ASP 협업 프로토콜 5개 · 유지보수 계약 0.5M→0.25M+별도 프로젝트 구조까지 4년+ 안정 운영의 기반을 만들었습니다.

Case 02 · 세인홈시스 · 2021.02 – 2022.02 · 1년

제조사 ERP·CRM·FSM 통합

ERP-CRM 통합과 FSM 모바일 앱을 동시 진행하고, 촉박한 일정 속에서 핵심 기능 선 개발(워터폴) 후 고도화 전략으로 ROI를 확보했습니다.

가전 제조사의 ERP·CRM·재고·민원이 분리·수기 관리되던 상태에서 30명+ TF를 총괄해 OMS/WMS/TMS 어드민과 FSM 앱 범위를 SRS·메뉴/데이터셋·마이그레이션 수식·테스트 시나리오로 구체화했습니다. QR/GPS 기반 방문 확인, 기사 배정, 알림톡, 현장 수불, 설치/수리 흐름을 디지털화하고, APP/CRM/DB 서버 운영·일일 백업·장애 처리·원본소스 현행화를 포함한 1년 유지보수 계약을 직접 협상·체결했습니다. FSM 동선 최적화 노하우는 이후 보험 설계사앱(SFA)으로 이식됐습니다.

Case 03 · 토스인슈어런스 · 2024 – 2025 · 3개월

CRM v2 재구축

사용률 하락을 화면 문제가 아니라 영업 데이터 인프라 리스크로 재정의하고, 187만/4.2만 분석으로 가설을 재검증했습니다.

CRM 사용률 95%→64.5% 하락은 단순 UX가 아니라 고객정보 유출·상담 퍼널 분석 공백·DB 활용 로드맵 붕괴가 맞물린 영업 데이터 인프라 리스크였습니다. CRM을 잘 쓰는 설계사의 실제 사용 방식·고객 데이터 1만 건·CX 상담 내역을 확인하고, 상담신청→버퍼계정→설계사배정→고객컨택→ERP계약 5단계 퍼널을 정의했습니다. 상담 DB 187만 건, 계약 DB 4.2만 건 분석으로 조직별 계약율과 리드타임 가설을 재검증한 뒤 30명+ TF로 3개월 안에 릴리즈했습니다.

Case 04 · 토스인슈어런스 · 2022 – 현재

정산 CMS 자동화 + ASP 종료 로드맵

정산 자동화와 외주 의존 종료를 별도 트랙이 아니라 한 로드맵으로 봤습니다. ASP→ERP 완전 내재화 단계로 분해했습니다.

월 100억 원 이상 수수료 정산이 엑셀 수작업 기반, ERP-CMS-SAP 데이터 싱크가 비표준, 5년간 외주 구조적 문제가 ASP 의존을 만성화시킨 상태였습니다. ERP-CMS-SAP 데이터 싱크 구조 설계 · 수수료율 검증 로직 3개(순번 고정 / 비교 / 합산 100%)로 휴먼에러 시스템 차단 · 5년간 외주 증적 정리로 ASP 종료와 소스코드 매입 협상 근거 확보 · 지넥슨 ASP 종료 후 내부 개발 채널 기반 이슈 접수·긴급 대응·현업 공지 체계로 운영 전환.

Case 05 · 토스인슈어런스 · 인켑 + AI UW · 2024 – 현재

AI 도입 분기 운영

하나의 AI 트랙으로 묶지 않고, 인켑은 IT Manager 자동화로, AI UW 센터는 외부 설계사용 판단 지원 PoC로 분리해 관리했습니다.

AI 활용 요구가 여러 부서에서 동시에 발생하는 상황에서 트랙을 분리했습니다. 인켑은 Slack 연동·데이터셋 오류 수정·웹 확장 검토·관리자 권한 정리, 수수료·계약팀 응대 확장 검토. AI UW 센터는 국내 6개 AI 솔루션 컨택·3개사 체크·로민 등 후보 검토 후 올거나이즈 선정 · 약관·인수기준표·상품 FAQ를 RDB + 벡터DB 하이브리드 Agent로 구성 · 가입 가능 여부 판단 정확도 80% 이상 기준 · 보안 검수 · PoC 진행 조건 정리.

04Career Arc

도메인은 더존 키컴 · 집닥 · 세인홈시스 · 토스인슈어런스로 바뀌었지만, 끝까지 굴려내는 방식은 이어졌습니다.

더존 키컴2017–2018

회계 시스템·웹팩스·스마트UMS 사업기획. 손익 시뮬레이션, 고객 만족도 전수 조사·인터뷰, VIP 솔루션 PoC/런칭으로 회계·사업성 감각 형성.

집닥2019–2020

인테리어 플랫폼 BM 재설계. CAC·LTV·ROI 분석으로 만성 적자 진단, 민원 1,010건·계약 5,388건 분석으로 분류 체계 재설계, 부분공사 16개 단가 구조 설계. BEP 달성 기여.

세인홈시스2021–2022

30명+ TF로 제조 ERP·CRM·FSM 통합과 QR/GPS 기반 모바일 앱 자체 구축. CS 민원 30% 감소. 1년 유지보수 계약 직접 협상.

토스인슈어런스2022–현재

60명 TF로 보험 ERP 0→1, 사용자 3,500명·계약 60만 건 운영 무중단 오픈. CMS·CRM v2·설계사앱·AI(인켑·AI UW 센터)를 보험 운영 플랫폼으로 확장. ASP 협업 프로토콜이 SAP·영선아이티에 재사용.

05이런 역할에 맞습니다

제 다섯 단계 실행 패턴이 가장 잘 풀리는 자리 여섯 가지입니다.

01

복잡한 멀티 시스템 0→1의 프로그램 오너

RFP·SRS·WBS·외주·인프라·보안까지 직접 설계하는 자리

02

외주·내부 R&R + 레거시 전환 리드

ASP/유지보수 계약 구조 재설계, 외주 협업 프로토콜 표준화

03

운영 데이터·사용자 행동을 기술 의사결정 근거로 보는 TPM

일정 관리가 아니라 가설 재검증과 우선순위 재정렬까지

04

AI 업무 시스템화

인켑·AI UW 같이 목적별 데이터·권한·품질·검증 루프 설계가 필요한 자리

05

보험·금융·제조 도메인 백오피스 플랫폼

ERP/CRM/CMS·정산·SAP 연동·규제 대응이 함께 굴러가야 하는 환경

06

실행 인프라 빌더

회의체·보고·장애 대응·유지보수 기준을 한 시스템으로 묶어 운영 자산화

06요약

복잡한 운영을 제품 시스템으로 바꾸는 9년차 Technical Program Manager.

보험·제조·핀테크 도메인에서 ERP·CRM·CMS·FSM·정산 시스템을 다섯 단계의 실행 패턴(프로그램 구조화 → 의존성 매핑 → 외주·내부 R&R → 리스크·장애 통제 → 운영 자산화)으로 0→1 굴려왔고, 외주 협업 프로토콜과 표준 아키텍처를 후속 프로젝트로 전이시켜 조직 자산으로 남겼습니다.
