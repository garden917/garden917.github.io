Summary

여러 조직의 업무·데이터·권한 흐름이 얽힌 문제를 제품 요구사항과 실행 기준으로 정리해 온 9년차 TPM/PM입니다.

레거시 시스템 전환, 수수료 정산 자동화, 보안 정책 도입처럼 비즈니스 운영과 기술 제약이 충돌하는 문제에서 제품 범위와 의사결정 기준을 세워 왔습니다. 일정·범위 관리는 실행 장치로 쓰되, 핵심은 업무 모델·데이터 흐름·보안 리스크를 제품 판단으로 바꾸는 데 둡니다.

불명확한 요구를 프로그램의 목적·범위·리스크·성공 기준으로 구조화하고, 외주사와 내부 조직이 같은 기준으로 움직일 수 있는 실행 체계를 만듭니다. 보안·개인정보·권한 제약은 출시 가능한 기준으로 구체화하고, 출시 이후에도 장애·정책·권한·데이터 품질을 운영 범위로 관리합니다.

핵심 역량

1. 기능 현황을 사업·사용자·데이터 관점에서 바라보고 문제를 정의합니다

CRM 시스템 사용률 95%→64.5% 하락 원인을 단순 UI 사용성 문제가 아니라 고객정보 유출, 상담 퍼널 분석 공백, DB 활용 로드맵 붕괴로 이어지는 영업 데이터 인프라 리스크로 정의하고, CRM v2 리빌드 후 사용률을 90%까지 회복했습니다. 단순 기능 개선보다 사업 손실, 사용자 행동, 데이터 흐름을 함께 보고 문제를 정의합니다.

2. 대형 프로젝트의 목적과 실행 기준을 정리하고 여러 조직이 같은 기준으로 움직이도록 실행 체계를 구축합니다

제품 도입 목적, 시급도별 구현 범위, 1차 오픈 범위, 프로젝트 TF 구성, 보안·운영 우선순위를 정리했습니다. 이후 RFP, SRS, WBS 기반 주간보고, 시스템 전환 데이터 마이그레이션 기준, UAT/오픈 기준, 장애 대응 체계를 구성해 60명 규모 TF가 같은 기준으로 움직이도록 총괄 PM 역할을 수행했습니다.

3. 업무 모델과 기술 제약을 제품 의사결정 기준으로 바꿉니다

토스인슈어런스 ERP/CMS에서는 수기 정산·환수·보안·권한 문제를 기능 요청이 아니라 운영 모델과 데이터 정합성 문제로 재정의했습니다. 세인홈시스 제조 ERP(OMS/WMS)·CRM·FSM에서는 현장 설치·수불·CS 흐름을 SRS, 메뉴·데이터셋, UAT 기준으로 구체화해 제품 실행 기준으로 만들었습니다.

4. 서비스 플로우를 퍼널과 ROI 관점으로 분석해 제품·운영 판단으로 연결합니다

다양한 도메인의 서비스 플로우를 리드 인입→유효 리드→상담/매칭→계약→운영 품질 단계로 구분하고, 전환율·리드 품질·운영 비용을 함께 봅니다. 집닥에서는 CAC·LTV·ROI, 파트너별 전환율, 계약·민원 데이터를 분석해 고객 DB 등급제와 파트너 등급제를 설계하고, 부분시공 상품/프로모션 방향을 정의했습니다. 토스인슈어런스에서는 CRM/ERP/DW 리드타임 지표로 유효 퍼널 46.7%, 30일 골든타임 70%를 확인해 고객 컨택 타이밍과 후속 분석 로드맵으로 연결했습니다.

5. 제품 릴리즈 이후 운영 기준까지 프로젝트 완료 범위로 봅니다

시스템 오픈을 프로젝트 종료로 보지 않고, 실제 운영에서 안정적으로 굴러가는 상태까지 제품/프로그램 범위로 관리합니다. 장애 대응, 외주 유지보수, Change Request, 현업 공지 기준을 정리하고 운영 과정의 개선 요청을 후속 범위·우선순위로 연결했습니다.
